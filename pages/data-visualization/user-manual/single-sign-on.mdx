# Signel sign-on (SSO)

Single sign-on (SSO) is a Memgraph enterprise feature that provides authorization and authentification to your database.

Memgraph supports two types of SSO methods - **OAuth 2.0** and **SAML**, and two identity
providers - **Microsoft Azure (Entra ID)** and **Okta**.
To set up SSO on Memgraph Lab you will have to configure the appropriate environment variables.

<Callout>

Single sign-on is not supported on the desktop version of Memgraph Lab.

</Callout>

## OAuth 2.0

The setup of OAuth 2.0 is done by setting two pairs of different environment variables.
One set is for Microsoft Entra ID, and a separate set of environment variables should be assigned for Okta.

### Entra ID

To set up OAuth 2.0 for Microsoft Azure,
[follow the official guide](https://learn.microsoft.com/en-us/entra/architecture/auth-oauth2#implement-oauth-20-with-microsoft-entra-id) and
[register an application](https://learn.microsoft.com/en-us/entra/identity-platform/quickstart-register-app).

To enable OAuth 2.0 for the Microsoft Entra ID identity provider, make sure to assign the
following environment variables when hosting Memgraph Lab:

```bash
AUTH_OAUTH_ENTRA_ID_IS_ENABLED=true
AUTH_OAUTH_ENTRA_ID_DISPLAY_NAME='MS Entra ID'
AUTH_OAUTH_ENTRA_ID_AUTHORIZATION_URL=
AUTH_OAUTH_ENTRA_ID_TOKEN_URL=
AUTH_OAUTH_ENTRA_ID_USER_INFO_URL=
AUTH_OAUTH_ENTRA_ID_CLIENT_ID=
AUTH_OAUTH_ENTRA_ID_CLIENT_SECRET=
AUTH_OAUTH_ENTRA_ID_CALLBACK_URL=.../auth/providers/oauth-entra-id/callback
AUTH_OAUTH_ENTRA_ID_SCOPE=openid
```

The empty values can be found on the Azure dashboard under the created app registration.

### Okta

To set up OAuth 2.0 for Okta, follow the [official guide](https://developer.okta.com/docs/guides/implement-grant-type/authcode/main/).

To enable OAuth 2.0 for the Okta provider, make sure to assign the
following environment variables when hosting Memgraph Lab:

```bash
AUTH_OAUTH_OKTA_IS_ENABLED=true
AUTH_OAUTH_OKTA_DISPLAY_NAME='MS Entra ID'
AUTH_OAUTH_OKTA_ISSUER=xxx.okta.com
AUTH_OAUTH_OKTA_AUTHORIZATION_URL=https://${AUTH_OAUTH_OKTA_ISSUER}/oauth2/v1/authorize
AUTH_OAUTH_OKTA_TOKEN_URL=https://${AUTH_OAUTH_OKTA_ISSUER}/oauth2/v1/token
AUTH_OAUTH_OKTA_CLIENT_ID=
AUTH_OAUTH_OKTA_CLIENT_SECRET=
AUTH_OAUTH_OKTA_CALLBACK_URL=xxx/auth/providers/oauth-entra-id/callback
AUTH_OAUTH_OKTA_SCOPE=openid
```

The empty values can be found on the Okta admin panel dashboard under the created app.
